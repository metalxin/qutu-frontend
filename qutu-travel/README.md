# 趣途云迹 (qutu-travel)


## 基于Spring Cloud微服务架构的智能旅行规划系统


## 一、项目介绍
### 1.1 项目概述
**趣途云迹** 是基于 **PIG 企业级微服务快速开发平台** 深度定制开发的智能旅行规划系统，本系统以Spring Cloud微服务生态为核心技术底座，整合智能行程规划、景点智能推荐、旅行攻略管理、用户权限控制等核心业务能力，打造一套高可用、高拓展、智能化的旅行行业解决方案。

系统兼顾企业级项目的安全性与毕业设计的实用性，既保留PIG平台原生的RBAC权限模型、OAuth2.0+Spring Authorization Server安全认证、分布式部署能力，又针对旅行场景定制开发智能行程生成、景点信息管理、多端适配等业务功能，是一套完整的前后端分离微服务项目。

### 1.2 核心特点
- 🚀 双架构支持：同时兼容「微服务部署」和「单体部署」，毕业设计本地开发首选单体模式，便捷高效
- 🔒 企业级安全：基于OAuth2实现统一授权认证，精细化RBAC权限控制，防XSS注入、密码加密存储，保障系统安全
- ✨ 智能核心：内置行程规划算法，根据用户出行目的地、游玩天数、出行偏好，自动生成个性化最优旅行行程
- 🎯 全栈技术：主流前后端分离技术栈，后端微服务解耦设计，前端自适应PC/移动端，易开发、易维护、易拓展
- 📚 功能完备：覆盖旅行场景全业务闭环，包含用户管理、景点管理、攻略发布、行程规划、收藏导出等核心功能
- 🛠️ 开发友好：内置代码生成器、接口文档、系统监控，大幅提升开发效率，完美适配毕业设计开发流程

## 四、系统核心功能
### ✅ 前台用户端（游客/普通用户）
1. 用户基础功能：账号注册、账号登录、个人信息修改、密码找回、退出登录
2. 智能行程规划：输入目的地、出行天数、出行偏好（自然风光/人文历史/美食探店），自动生成最优旅行行程表
3. 景点信息服务：景点列表查询、景点详情展示（地址/门票/开放时间/景点介绍）、景点收藏、景点评价
4. 旅行攻略中心：精选攻略展示、攻略分类查询、攻略收藏、攻略搜索、热门攻略推荐
5. 行程管理中心：我的行程查看、行程编辑修改、行程导出、历史行程记录、行程收藏夹

### ✅ 后台管理端（管理员角色）
1. 系统权限管理：用户账号增删改查、角色分配、菜单权限配置、部门管理
2. 景点信息管理：景点数据维护、景点分类管理、景点图片上传、热门景点推荐配置
3. 攻略内容管理：旅行攻略审核、攻略发布、攻略下架、攻略分类维护
4. 行程模板管理：定制化行程模板配置、热门行程推荐、行程数据统计
5. 系统基础配置：轮播图管理、系统公告发布、网站信息配置
6. 系统监控中心：接口访问日志、用户操作日志、服务运行状态监控

## 五、快速启动教程
### 5.1 本地开发环境要求 (毕业设计首选)
- JDK 版本：JDK 17 及以上
- Maven 版本：3.8.0 及以上
- 数据库版本：MySQL 8.0 及以上
- 缓存服务：Redis 6.0 及以上
- 前端环境：Node 18+ 、npm 9+

### 5.2 后端启动步骤
1. 本地启动MySQL和Redis服务，创建数据库 `qutu_travel`
2. 执行项目根目录下 `sql` 文件夹内的初始化脚本 `qutu_travel.sql`
3. 用IDEA打开后端项目，修改 `application.yml` 配置文件中的数据库、Redis连接信息
4. 启动优先级：先启动 `qutu-register` (Nacos) → 再启动 `qutu-boot` 单体模块
5. 启动成功后，后端接口根路径：`http://localhost:9999`
6. 接口文档访问地址：`http://localhost:9999/doc.html` (账号/密码：admin/123456)

### 5.3 前端启动步骤
1. 用HBuilderX/Vscode打开 `qutu-ui` 前端工程
2. 执行命令安装依赖：`npm install`
3. 启动前端项目：`npm run dev`
4. 前端访问地址：`http://localhost:5173`
5. 测试账号：管理员(admin/123456)、普通用户(user/123456)

### 5.4 Docker 一键快速体验
```shell
# 环境要求：本机可用内存 ≥ 4G
curl -o docker-compose.yaml https://try.qutu.sdstudio.cn
# 执行启动命令，等待3-5分钟即可访问
docker compose up -d
```

## 六、项目设计说明
### 6.1 架构设计理念
本项目基于成熟的 PIG 微服务开发平台进行业务定制，核心设计思路为「基础能力复用、业务能力定制」，无需从零开发权限、认证、网关等基础模块，将开发重心聚焦于旅行核心业务实现，既保证了系统的稳定性，又提升了开发效率，完全符合毕业设计的开发周期与技术要求。
### 6.2 核心技术亮点
微服务解耦：所有业务模块独立拆分，降低模块间耦合度，便于功能拓展与维护
智能算法集成：行程规划模块整合用户多维度需求，生成精准的个性化旅行方案
安全机制完善：全链路的安全防护，包含接口权限校验、密码加密、XSS 过滤、SQL 防注入
前后端分离：前端与后端完全解耦，可独立部署、独立迭代，适配多终端访问需求
